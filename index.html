Touch Only What Needs Touching: Novel Evaluation Protocol on Manipulation Abilities of Text-Guided Manipulation Models

Abstract

Recent advancements in multi-modal generative models have led to the development of text-guided image editing models, which have gained popularity for creative and artistic purposes. Well-manipulated samples must follow the text prompt while maintaining the original image's essence. However, existing metrics determine only one of these two aspects. In this paper, we propose a novel metric AugCLIP that simultaneously evaluates editing and preserving according to the relative importance of the visual factors created by a large language model. We also propose a new benchmark dataset, SameSwap, to evaluate the ability of the manipulation model to identify the area to be manipulated, constructed with four different situations. Finally, we provide an analysis of different kinds of manipulation models regarding visual factors.

Sameswap


We introduce SameSwap, an innovative collection of image-text prompt pairs designed to describe desired image transformations. SameSwap is tailored to deliberately create challenging scenarios where the objects within an image either share the same noun (referred to as "same noun") or exhibit the same attributes (referred to as "same attr"). In developing target text prompts for SameSwap, we offer two distinct cases to accommodate a wide range of creative possibilities: "swap noun" and "swap attr." Consequently, SameSwap encompasses a total of four unique scenarios: same attr swap attr, same attr swap noun, same noun swap attr, and same noun swap noun. SameSwap can determine whether the manipulation model has properly manipulated the targeting part because it intentionally created a situation where the two objects included in the image were confused.

Dataset Generation Process

1. Image Selection from COCO Dataset
The primary selection criterion involves identifying the two prominent objects, A and B, that collectively occupy over 10% of the total image area. We ensure that their overlap, determined by the Intersection over Union (IoU) metric, remains below 0.1. This process is done based on annotation from GQA dataset.
2. Image Filtering
Owing to potential inaccuracies in attribute annotations within the GQA dataset, we enforce a filtration process. Images are retained for further analysis only if the CLIP similarity between the annotation and the object region surpasses the initial quarter (0.217) of the mean similarity values obtained from all candidate images. Further refining the dataset, we filter out images with potential object confusion. This involves assessing the CLIP similarity between the two identified object regions. We only retain images where the similarity falls below the median value, 0.746.
3. Text Prompt Generation: Noun swap
Leveraging the class hierarchy from the OpenImages V7 dataset, we randomly select a sibling category. To address contextual disparities as shown in above image, we employ the captioning model ExpansionNetv2 to extract additional information in the form of captions. Subsequently, we gauge the probability of a randomly substituted word using Language Model (LLM) and select a noun with high probability.
3. Text Prompt Generation: Attribute swap
We created an extensive list of possible attributes that could be matched with the given noun by gathering all possible attribute instances from GQA dataset. Based on this list, we construct candidate attribute list by excluding those present in the original object. Among these candidates, we swap with attributes that have a concreteness rating score higher than 4.3. Also, to avoid minor change, we select an attribute that has lowest CLIP similarity between the target prompt and the original image.
